# Frontend Patterns

Use this skill when the task is about frontend implementation structure rather
than visual product polish: React/Next.js components, hooks, state, forms,
performance, error boundaries, animation primitives, or accessibility behavior.

For production visual design, layout quality, interaction states, and
design-system fit, use `frontend-ui-engineering` first and load this skill only
for implementation details.

## Workflow

1. Identify the specific frontend concern: component composition, hook design,
   state, performance, forms, error boundaries, animation, or accessibility.
2. Inspect nearby app conventions before introducing a pattern.
3. Prefer the simplest local pattern that solves the current problem.
4. Load `references/full-guide.md` only for detailed examples or checklists.
5. Verify with the relevant project checks and, for browser-visible behavior,
   a real browser or screenshot check when available.

## Reference

- `references/full-guide.md` preserves the original detailed guide with
  examples for composition, compound components, render props, hooks, state,
  memoization, lazy loading, virtualization, forms, error boundaries, animation,
  keyboard navigation, and focus management.
